# Replication archive — Duration-Dependent Flood Depth-Damage Function

This archive contains all code, data, and paper sources required to reproduce the
results of the paper *"A duration-dependent flood depth-damage function calibrated
using FEMA NFIP claims from three U.S. hurricanes"*.

**Snapshot date:** 2026-05-09 (Europe/Madrid)
**Calibration claims:** 295,637 (down from a Mar-2026 snapshot of 317,943 — see
`params/PROVENANCE.md` for the full audit trail).

## Headline canonical fit (2026-05-09 FEMA snapshot, 295,637 valid / 295,400 used)

| Parameter | Value  | Note                                     |
|-----------|-------:|------------------------------------------|
| B         | 5.6473 | Steepness                                |
| Q₀        | 3.4502 | Initial threshold asymmetry              |
| a₀        | 0.5417 | Inflection at τ = 0                      |
| ν₀        | 2.6091 | Derivative asymmetry at τ = 0            |
| λ_a       | 0.4893 | **Inflection shift rate (significant)**  |
| λ_Q       | ≈ 0    | Onset speed modifier (not significant)   |
| δ_ν       | ≈ 0    | Asymmetry growth modifier (not significant) |

**Headline predictions** (these reproduce the paper-text values exactly):

| h, τ              | DR (this fit) | DR (paper-text) |
|-------------------|--------------:|----------------:|
| 1 m, τ = 0 (instantaneous) | 0.250 | 0.25 |
| 1 m, τ = 36 h (Sandy-like) | 0.353 | 0.35 |
| 1 m, τ = 288 h (Harvey-like) | 0.656 | 0.64 |
| Amplification 0 → 288 h     | 2.63× | 2.6× |

The four base Richards parameters (B, Q₀, a₀, ν₀) differ from the paper text along the model's identifiability ridge — multiple parameter tuples produce equivalent D(h, τ) curves. Practitioners should rely on D(h, τ) predictions; see `params/PROVENANCE.md` for the full identifiability discussion.

**Qualitative finding** (consistent with the published paper version): duration
affects damage *only* through inflection-point shift (λ_a ≈ 0.6); curve-shape
parameters λ_Q and δ_ν are not distinguishable from zero given three event-level
duration values.

## How to reproduce

### Option A — refit on this archive's frozen data (deterministic, ~1 min)

```bash
pip install -r requirements.txt
PYTHONPATH=code python code/notebooks/unified_calibration.py
```

This reads the frozen `data/processed/fema_*_calibration.csv` files (already filtered
from the 2026-05-09 OpenFEMA snapshot) and refits using `differential_evolution`
with `seed=42`. Output:

* `params/unified_params.csv` — same numbers as `unified_params_canonical.csv`
  to 4 decimal places (last-decimal differences possible due to BLAS thread
  ordering).
* `figures/10_unified_calibration.png` — the same figure as in the paper, plus
  binned residuals.

### Option B — re-pull from the OpenFEMA API (10–30 min, snapshot-dependent)

```bash
pip install -r requirements.txt
PYTHONPATH=code python code/fetch_fema.py     # downloads ~445k raw claims
PYTHONPATH=code python code/notebooks/unified_calibration.py
```

`fetch_fema.py` writes fresh `data/raw/*.csv` and `data/processed/*.csv`. Counts
will differ slightly from this archive because FEMA continues to backfill
historical claims (mostly with missing fields that fail our filter).

### Option C — full reproduce.sh (Option A by default)

```bash
./reproduce.sh
```

## Filter applied (matches paper Table 1 caption verbatim)

A claim is **valid** if and only if:

1. `waterDepth > 0` (positive recorded flood depth)
2. `total_payout > 0` (building + contents payout combined)
3. `total_coverage > 0` (positive insurance — denominator non-zero)
4. `damage_ratio = total_payout / total_coverage ∈ (0, 1.5]`, capped at 1.0

There are **no other filters** in the data pipeline (this is a fix relative to
earlier code which silently applied `total_coverage > $1,000` and `depth ≤ 10 m`
caps not described in the paper text — both filters have been removed in this
archive).

Per-event depth caps are applied at the *calibration* step (not the data filter)
to match each event's data range: Sandy/Harvey at 4 m (99.99% of valid claims for
each event; Sandy 99.5 pct = 2.4 m, Harvey 99.5 pct = 1.1 m), Katrina at 10 m
(99.93%; Katrina 99.5 pct = 7.3 m). This retains 99.91% of valid claims overall
(295,400 of 295,637) while excluding the small extreme-outlier tail above each
event's data range. An earlier uniform 6 m cap discarded the Katrina levee-breach
tail (1,216 claims, 0.9% of valid Katrina) — the data subset that most
distinguishes Katrina from the shorter-duration events. See `params/PROVENANCE.md`
for the cap-regime sensitivity analysis.
